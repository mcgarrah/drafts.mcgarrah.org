Jekyll Enhancements Without Plugins

By Michael McGarrah · May 10, 20262 min read

A collection of Jekyll enhancements that can be implemented without plugins — useful for GitHub Pages compatibility and reducing dependency complexity. Many of these were evaluated during the build-out of this blog, with some adopted and others deferred.

Jekyll Without Plugins Reference

The Jekyll Codex maintains an excellent collection of plugin-free implementations. Here are the ones most relevant to this blog:

Search

  • Google Custom Search integration — Implemented on this blog via the Adding Google Custom Search post.

Comments

Several approaches were evaluated before settling on Giscus:

  • Jekyll Codex comments overview
  • GDPR-compliant comments — Relevant to our GDPR implementation
  • GitHub Issues-based comments — Evaluated but Giscus (GitHub Discussions) was preferred
  • Giscus self-hosting — Reference for future consideration
    • Bartosz Gorka’s Giscus setup guide — Used as reference during implementation
  • Utterances — Predecessor to Giscus, uses GitHub Issues instead of Discussions
  • GitHub Issues API approach — Lower-level alternative

UX Enhancements

  • Reading time indicator — Implemented on this blog
  • Cookie consent — Evaluated; custom GDPR implementation was built instead
  • Breadcrumbs — Not yet implemented
  • SEO without plugin — Using jekyll-seo-tag plugin instead
  • Share buttons — Not yet implemented
  • Text expand/collapse — Not yet implemented

Jekyll Plugin Development

For cases where plugins are needed:

  • Your First Plugin guide — Official Jekyll docs
  • reading_time plugin — Alternative to without-plugin approach
  • jekyll-analytics — Multi-provider analytics
  • jekyll-seo-tag — In use on this blog
  • jekyll-seo-gem — Alternative SEO approach
  • jekyll-admin — Web-based admin interface
  • jekyll-google_search_console_verification_file — GSC verification
  • jemoji — GitHub-style emoji support

Template Branch for Reusable Starter

There is a clean branch of this repository that was started as a reusable template for others to use as a starting point with all the blog features (GDPR, SEO, Giscus, Mermaid, etc.) but with personal content removed. The upstream branch tracks the original forked theme.

  • Compare branches: upstream…clean
  • The divergence from the original fork has grown significant, making upstream contributions difficult
  • The original plan was to break changes into separate merge requests:
    1. Pagination with archives in page counts
    2. Jekyll upgrade to 4.3+
    3. New GitHub Action Workflow
    4. Embedded features for sizing images
    5. Conditional Google Analytics and Google AdSense
    6. RSS Sitemaps
