What Needs Attention

Experience Descriptions Are IC-Voiced

The biggest gap is in the experience section. Every role description reads as what you did, not what you led, decided, or enabled. This is the same Pattern #1 from PERSONA.md (“Lead with Why, Not What”) applied to job descriptions.

Envestnet (current role):

“Building large scale cloud platforms for financial services and data science initiatives.”

This is IC language. “Building” = doing the work. For SVP positioning, reframe as:

“Leading cloud platform architecture for financial services, designing the infrastructure that supports data science initiatives across the organization. Driving automation-first strategy integrating AWS cloud and on-premises data center infrastructure.”

Blue Cross NC:

“Building a data science platform incorporating a data-warehouse, data-lake and data-execution environment”

Reframe to surface the leadership and impact:

“Architected and led development of the enterprise data science platform — integrating data warehouse, data lake, and containerized execution environment — that powered production deep learning models for healthcare predictions. Platform supported the CarePath initiative (ISSIP Excellence Award).”

USPS:

“Engaged in advanced analytics with multiple platforms.”

“Engaged in” is passive. Reframe:

“Administered a 25-node SAS Viya in-memory analytics cluster (26TB RAM) linked to a Hadoop data lake approaching 1PB. Operated in a high-security federal environment processing terabyte-scale datasets for advanced analytics and machine learning.”

NC DIT: This one is already the strongest — “Accountable for developing, maintaining and overseeing” is leadership language. Minor polish only.

NC DOR / ETAAC: These are already strong. The ETAAC description is excellent — “reports to Congress annually” is a credibility signal most candidates can’t match.
